üìã Overview

This project builds a classification model to predict whether a Pok√©mon is legendary based on its combat statistics and abilities. The process involved data integration from multiple sources, extensive cleaning and preprocessing, exploratory data analysis to understand key characteristics, and training an imbalanced-aware Random Forest Classifier.

üìä Data Sources

Two datasets from KaggleHub were utilized:

  • ulrikthygepedersen/pokemon-stats: Providing core combat statistics (HP, Attack, Defense, etc.), types, and legendary status.
  • tanishksharma9905/pokemon-data-csv: Offering additional details like height, weight, and a structured 'abilities' column.

üßπ Data Cleaning & Preparation

  • Initial Loading: Both datasets were loaded into Pandas DataFrames (df_pokemon_stats and df_pokemon_data).
  • Name Standardization: Pok√©mon names were standardized to lowercase (name_lower) and special characters were mapped (e.g., 'nidoran‚ôÄ' to 'nidoran-f') for consistent merging.
  • Form Filtering: Various temporary battle forms (Mega, Gigantamax, Primal) and cosmetic variants were filtered out to retain base forms and meaningful variants across both datasets, ensuring more accurate matching.
  • Dataset Merging: The two cleaned datasets were merged using an inner join on the name_lower column. This resulted in merged_df, containing only Pok√©mon present in both original sources with comprehensive features.
  • Handling Unique Rows: Pok√©mon present in one dataset but not the other were identified and exported to separate JSON files. These were omitted from the main analysis to maintain data completeness.
  • Ability Column Split: The abilities column (which contained string representations of lists or comma-separated strings) was parsed into main_ability and sub_ability columns.
  • Unit Conversion: 'height' was converted from decimeters to centimeters (multiplied by 10) and 'weight' from hectograms to grams (multiplied by 100).
  • Unnecessary Column Removal: Columns such as name_lower, name_data, base_experience, types, moves, and stats were dropped to focus on relevant numerical and categorical features.

üîç Exploratory Data Analysis (EDA)

  • Legendary Pok√©mon Type Analysis: Identified Psychic, Dragon, and Steel as the most common types among legendary Pok√©mon.
  • Data Imbalance Visualization: A pie chart clearly showed a significant class imbalance: approximately 92.5% Non-Legendary vs. 7.5% Legendary Pok√©mon.
  • Mean Stats Comparison: Analysis confirmed that legendary Pok√©mon consistently have higher average combat stats across all categories compared to non-legendary Pok√©mon, with 'Sp. Attack' and 'Sp. Defense' showing the largest differences.
  • Stat Distribution (Box Plots): Visualized stat distributions for both groups, reinforcing the higher stat averages for legendaries and highlighting 'Sp. Attack' and 'Sp. Defense' for their strong separation.
  • Feature Correlation: A heatmap of combat stats showed moderate correlations, indicating features are sufficiently distinct for modeling without severe multicollinearity.

ü§ñ Model Training

  • Feature Scaling: Numerical features (hp, attack, defense, sp_attack, sp_defense, speed) were scaled using StandardScaler to normalize their range, resulting in X_scaled_df.
  • Data Splitting: The dataset was split into 80% training and 20% testing sets using train_test_split with stratify=y to maintain the class ratio in both sets.
  • Class Imbalance Handling: SMOTE (Synthetic Minority Over-sampling Technique) was applied only to the training data, creating a balanced training set with a 50/50 split of legendary and non-legendary Pok√©mon.
  • Model Initialization & Training: A RandomForestClassifier was initialized with n_estimators=100, class_weight='balanced', random_state=42, and max_depth=None. The model was then fitted on the SMOTE-resampled training data.

üìà Model Evaluation

The Random Forest model's performance was evaluated on the unseen test data using various metrics:

  • Accuracy: Achieved an overall accuracy of 94.12%.
  • Classification Report:
    • Non-Legendary: Precision 0.96, Recall 0.98, F1-score 0.97
    • Legendary: Precision 0.67, Recall 0.46, F1-score 0.55
    • The model is better at identifying non-legendaries than actual legendaries.
  • Confusion Matrix:
    • True Negatives: 154 (Correctly predicted Non-Legendary)
    • False Positives: 3 (Misclassified Non-Legendary as Legendary)
    • False Negatives: 7 (Misclassified Legendary as Non-Legendary)
    • True Positives: 6 (Correctly predicted Legendary)
  • AUC-ROC Score: 0.7567, indicating a fair ability to distinguish between classes.
  • Feature Importance: The most important features identified by the model were sp_defense, hp, sp_attack, and speed.

üíæ Model Persistence

To allow for future use and reproducibility:

  • The trained RandomForestClassifier model was serialized to legendary_classifier_v1.pkl.
  • The StandardScaler used for feature scaling was serialized to scaler.pkl.
  • The SMOTE transformer was serialized to smote_transformer.pkl.
  • Detailed model metadata, including training date, hyperparameters, dataset sizes, and final evaluation metrics, was saved to model_metadata.json.

‚ú® Key Takeaways

  • Legendary Pok√©mon have significantly higher stat averages, especially in Special Attack and Special Defense.
  • The 92.5% to 7.5% class imbalance was successfully mitigated using SMOTE during training.
  • The model achieves 94% overall accuracy with strong non-legendary classification.
  • Sp. Defense, HP, Sp. Attack, and Speed are the most influential features for legendary classification.
  • The trained model and scalers have been persisted for reproducible predictions on new Pok√©mon data.
